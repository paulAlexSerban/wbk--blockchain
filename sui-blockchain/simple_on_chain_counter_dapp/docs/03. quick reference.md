## ğŸ“ Common Command Patterns

### Full Deployment Flow
```bash
# 1. Build
cd simple_on_chain_counter_dapp__contract
sui move build

# 2. Check environment
sui client active-env
# Should show: testnet

# 3. Check balance
sui client gas

# 4. Deploy
sui client publish --gas-budget 100000000

# 5. Save output (Package ID, Transaction Hash)
```

### Full Testing Flow
```bash
# 1. Create counter
sui client call \
  --package <PACKAGE_ID> \
  --module simple_on_chain_counter_dapp__contract \
  --function create_counter \
  --gas-budget 10000000

# 2. Save Counter Object ID from output

# 3. Increment counter
sui client call \
  --package <PACKAGE_ID> \
  --module simple_on_chain_counter_dapp__contract \
  --function increment \
  --args <COUNTER_OBJECT_ID> \
  --gas-budget 10000000

# 4. Check counter state
sui client object <COUNTER_OBJECT_ID>
```

## ğŸ’¡ Tips

- **Gas Budget**: Set based on operation complexity
  - Simple calls: 10000000 (0.01 SUI)
  - Publishing: 100000000 (0.1 SUI)
  
- **Object IDs**: Always start with `0x` and are 32 bytes (64 hex characters)

- **Save Important IDs**: Keep track of:
  - Package ID
  - Object IDs (Counter, etc.)
  - Transaction Hashes
  
- **Environment Check**: Always verify you're on the correct network before deploying

- **Gas Management**: Request faucet tokens before running out
